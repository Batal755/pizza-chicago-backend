# Идеи по улучшению проекта

Сгруппировано по приоритету. Сверху — то, что даёт максимум пользы при разумных усилиях.

## 1. Сначала довести до «зелёного» (обязательно)

- **Запустить и проверить весь стек.** Код написан, но ни разу не собирался и не запускался.
  Первый шаг: `./scripts/setup.sh` → `./scripts/dev.sh`, прогнать `npm test` в каждом пакете,
  починить то, что вылезет (обычно версии пакетов и мелочи типов).
- **Playwright e2e** — сквозной тест в реальном браузере: открыть сайт → собрать корзину →
  оформить → оплатить (mock) → увидеть статус. Сейчас e2e есть только на уровне API (шлюз).
- **CI (GitHub Actions)** — на каждый пуш: lint + build + тесты всех пакетов. Защищает от регрессий.

## 2. Надёжность и безопасность (важно перед боем)

- **Rotating refresh-токены + выход со всех устройств** — хранить хеш refresh-токена в БД,
  отзывать при выходе. Сейчас refresh stateless.
- **Идемпотентность заказов и платежей** — ключ идемпотентности, чтобы двойной клик/повтор
  вебхука не создавал дублей.
- **Проверка подписи вебхука оплаты** — сейчас вебхук доверяет телу. Для боя — проверять
  подпись/IP ЮKassa.
- **Глобальный rate-limit и капча на формах** (регистрация, поддержка) — против спама.
- **Очередь событий вместо прямого gRPC-вызова уведомлений** (RabbitMQ/NATS/Kafka) — заказ
  не должен ждать почту; уведомления и аналитика — через события, с ретраями.
- **Healthcheck-эндпоинты** у каждого сервиса + readiness/liveness для оркестратора.
- **Централизованные логи и трейсинг** (OpenTelemetry + Jaeger) — видеть путь запроса через gRPC.

## 3. Продукт — то, что приносит деньги

- **Акции и промокоды** — скидки, «2+1», первый заказ дешевле. Главный драйвер заказов.
- **Бонусная программа** — кэшбэк баллами, возвращает клиентов.
- **Конструктор пиццы** — добавки за доплату, поднимает средний чек.
- **Статусы заказа в реальном времени** для клиента (готовится → в пути) через WebSocket/SSE.
- **Карта зон доставки + расчёт стоимости и времени** по адресу.
- **Отзывы на сайте** (подтягивать с 2ГИС) — соцдоказательство.
- **Telegram-бот** — заказы и уведомления прямо в Telegram (популярно в регионе).

## 4. Удобство разработки и эксплуатации

- **Монорепо-менеджер** (pnpm workspaces / Turborepo / Nx) — единая установка и сборка,
  общий пакет типов из `proto/` (codegen в TypeScript, чтобы не описывать интерфейсы руками).
- **Dockerfile на каждый сервис + единый `docker-compose` для всего стека** — запуск одной
  командой без локального Node.
- **Генерация TS-типов из .proto** (ts-proto) — убрать дублирование интерфейсов в клиентах.
- **Sentry** — сбор ошибок фронта и бэка в проде.
- **Админка: дашборд со статистикой** — выручка за день/неделю, топ-пиццы, средний чек.

## 5. Производительность

- **Кэш меню** (Redis) на шлюзе — меню меняется редко, а читается на каждом заходе.
- **Next.js: ISR/кэширование** страницы меню, оптимизация изображений (next/image + реальные фото).
- **Пагинация** заказов в админке, когда их станет много.

---

Рекомендуемый порядок: **1 → 2 → 3**. Сначала убедиться, что всё работает и безопасно,
потом наращивать продуктовые фичи под реальные продажи.
